## 1. GitHub에 새 저장소 만들기
- aba-geomdan 계정으로 로그인
- 새 저장소 이름: **geomdan-guide** (Public)
- README 등 추가 항목 체크하지 말고 그냥 생성

## 2. 이 폴더에서 (.env 먼저 확인)
- `.env` 파일에 관리자 비밀번호가 들어 있습니다 (현재: abageomdan1121).
- `.env` 는 git에 올라가지 않으므로 비밀번호가 공개 저장소에 노출되지 않습니다.
- 비밀번호를 바꾸려면 `.env` 의 `VITE_ADMIN_PASSWORD` 값을 수정하세요.

```bash
npm install
npm run build
git init
git add -A
git commit -m "초기 배포"
git branch -M main
git remote add origin https://github.com/aba-geomdan/geomdan-guide.git
git push -u origin main

# gh-pages 브랜치로 dist 폴더 배포
npm run deploy
```

## 3. GitHub Pages 설정
- 저장소 → Settings → Pages
- Source: **Deploy from a branch**
- Branch: **gh-pages** / (root) 선택 후 Save
- 1~2분 후 https://aba-geomdan.github.io/geomdan-guide/ 접속

## 4. 코드 수정 후 재배포
```bash
npm run build
npm run deploy
```

## 이번 버전에서 바뀐 점

### 중앙 저장(Supabase) — 어디서든 로그인 · 기기 무관 · 관리자 통합
계정과 가이드를 Supabase 중앙 DB에 저장합니다. 이걸 켜면:
- **어디서든 로그인** — 계정이 중앙에 있어 어느 기기·브라우저에서도 로그인됩니다.
- **본인 데이터 기기 무관** — 집/센터/휴대폰 어디서 만든 가이드든 본인 것이 다 보입니다.
- **관리자 통합 조회** — 관리자 페이지 → "전체 통합 보기"로 모든 선생님 가이드를 한 화면에서 봅니다(각 가이드에 작성자 표시).
- 비밀번호는 PBKDF2 해시로 저장됩니다(평문 저장 안 함).

이 기능은 `.env` 의 `VITE_SUPABASE_URL` 과 `VITE_SUPABASE_ANON_KEY` 를
둘 다 채웠을 때만 켜집니다. 비우면 기존처럼 기기별 localStorage 로 동작합니다.

#### Supabase 설정 방법

**1) 테이블 만들기**
- Supabase 대시보드(supabase.com) → 프로젝트(vdubgrxwijydwfabwpnk) 선택
- 왼쪽 메뉴 SQL Editor → New query
- 이 폴더의 `supabase/schema.sql` 내용을 전부 복사해 붙여넣고 Run
- 테이블 4개(gd_users, gd_guides, gd_favorites, gd_custom_templates)가 생성됩니다.

**2) API 정보 확인**
- 대시보드 → Project Settings → API 메뉴
- **Project URL** (https://vdubgrxwijydwfabwpnk.supabase.co) 복사
- **anon public** 키 복사 (service_role 키는 절대 쓰지 마세요)

**3) .env 에 입력**
```
VITE_SUPABASE_URL=https://vdubgrxwijydwfabwpnk.supabase.co
VITE_SUPABASE_ANON_KEY=eyJ...복사한_anon_키...
```

**4) 다시 빌드 & 배포**
```bash
npm run build
npm run deploy
```
이제 어느 기기에서 접속해도 같은 계정·데이터를 씁니다.
첫 실행 시 admin 계정이 DB에 자동 생성됩니다(.env 의 VITE_ADMIN_PASSWORD 사용).

#### 보안 참고
- 이 방식은 내부용(센터 선생님 전용)에 맞춰 anon 키로 DB에 접근합니다.
- anon 키는 공개돼도 무방한 키지만, 데이터 보호를 강화하려면 추후
  Supabase RLS 정책 또는 Edge Function 경유 접근으로 올릴 수 있습니다.
- service_role 키(관리자 키)는 앱·깃에 절대 넣지 마세요.

### AI 생성 + 템플릿 폴백 (합친 방식)
- 기본은 **AI가 주제마다 새 가이드를 작성**합니다.
- AI 호출이 실패하거나 AI를 설정하지 않으면 **자동으로 템플릿 폴백**합니다.
- AI는 Supabase Edge Function을 통해 호출되며 API 키는 함수에만 보관됩니다.

### AI 설정 방법 (Supabase Edge Function)

이미 통합 시스템에서 쓰시는 Supabase 프로젝트
(vdubgrxwijydwfabwpnk)에 함수 하나만 추가하면 됩니다.

#### 1) Supabase CLI 설치 (한 번만)
```bash
npm install -g supabase
supabase login
```

#### 2) 프로젝트 연결 (이 폴더에서)
```bash
supabase link --project-ref vdubgrxwijydwfabwpnk
```

#### 3) Anthropic API 키를 시크릿으로 등록
```bash
supabase secrets set ANTHROPIC_API_KEY=sk-ant-...여기에_실제_키...
```
- 키는 Anthropic 콘솔(console.anthropic.com)에서 발급합니다.
- 이 키는 함수 서버에만 저장되고 앱·깃에는 들어가지 않습니다.

#### 4) 함수 배포
```bash
supabase functions deploy generate-guide --no-verify-jwt
```
- `supabase/functions/generate-guide/index.ts` 가 배포됩니다.
- `--no-verify-jwt` 는 앱에서 인증 없이 호출하기 위함입니다.

#### 5) 배포된 함수 URL을 .env 에 입력
배포가 끝나면 함수 URL이 출력됩니다. 보통:
```
https://vdubgrxwijydwfabwpnk.supabase.co/functions/v1/generate-guide
```
이 값을 `.env` 의 `VITE_AI_ENDPOINT` 에 넣으세요.
```
VITE_AI_ENDPOINT=https://vdubgrxwijydwfabwpnk.supabase.co/functions/v1/generate-guide
```

#### 6) 다시 빌드 & 배포
```bash
npm run build
npm run deploy
```
이제 앱에서 AI 가이드가 생성됩니다.
(AI가 응답하지 않으면 조용히 템플릿으로 대체됩니다.)

#### 비용 참고
- 가이드 한 장마다 Anthropic API 호출 비용이 발생합니다(사용량 과금).
- 비용을 멈추고 싶으면 `.env` 의 `VITE_AI_ENDPOINT` 를 비우고
  다시 빌드·배포하면 됩니다. 그러면 템플릿만 사용합니다.

### 영역(주제 종류) 자동 분류 — 템플릿 폴백 품질
- 주제를 키워드로 5개 영역으로 자동 분류합니다:
  언어·요구하기 / 자조·일상생활 / 사회성·상호작용 / 학습·인지 / 도전행동 줄이기.
- AI 폴백 시에도 영역에 맞는 실천법·준비물·목표가 채워집니다.
- 입력 화면에서 자동 판별 결과가 보이고, 드롭다운으로 직접 바꿀 수 있습니다.

### 관리자 템플릿 편집 — 폴백 문구 맞춤
- 관리자 페이지 → "가이드 템플릿 관리" 버튼으로 진입.
- 행동 기능(5) × 연습 영역(5) 조합별 9개 항목 문구를 직접 수정·저장.
- 저장한 문구는 템플릿 생성 시 우선 적용됩니다
  (커스텀 > 영역 기본값 > 기능 기본값).
- `{name}` `{name_ga}` `{name_neun}` `{topic}` 자리표시자 자동 치환.

### 코드에서 기본 문구·프롬프트 수정 방법
- AI 프롬프트: `supabase/functions/generate-guide/index.ts` 의 `SYSTEM_PROMPT`.
- 템플릿 문구: `src/App.jsx` 의 `GUIDE_TEMPLATES`(기능별), `DOMAIN_FIELDS`(영역별).
- 영역 분류 키워드: `src/App.jsx` 의 `DOMAINS` 배열의 `keywords`.

### 보안·마감 처리
- 관리자 비밀번호를 빌드 환경변수(.env)로 분리, 화면 노출 제거
- 파비콘(연구소 로고), 페이지 제목·설명·OG 태그 추가
- 검색엔진 비노출(noindex) — 내부용 앱 보호
- GitHub Pages 새로고침/딥링크 404 처리 추가

## 주의
- 저장소 이름을 geomdan-guide 가 아닌 다른 이름으로 만들면,
  `vite.config.js` 의 `base`, `public/404.html`, `index.html` 안의
  `/geomdan-guide/` 경로를 모두 그 이름으로 바꿔야 합니다.
- 데이터는 각 사용자의 브라우저(localStorage)에 저장됩니다.
  기기·브라우저가 바뀌면 데이터가 공유되지 않습니다.
  여러 기기에서 공유가 필요하면 추후 Supabase 연동이 필요합니다.
